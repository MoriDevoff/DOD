# Django + PyInstaller для Windows

Ниже описан рабочий portable-вариант для проекта, который запускается одним основным `exe` файлом и открывает браузер автоматически.

## 1. Что получится

- Пользователь запускает один `launcher.exe`.
- Django-сервер поднимается локально на `127.0.0.1:8000`.
- Браузер открывается автоматически.
- SQLite лежит рядом с exe и работает локально.
- Фото и другие загруженные файлы сохраняются в `media` рядом с exe.
- Статика доступна из `static` рядом с exe.

## 2. Оптимальный способ упаковки

Для вашей задачи нужен именно `PyInstaller --onefile`, чтобы итогом был один исполняемый файл.

Почему так:

- это один файл для передачи;
- шаблоны и статика могут быть встроены внутрь exe;
- SQLite и `media` будут храниться рядом с exe и сохраняться между запусками.

`--onedir` тоже рабочий вариант, но если нужен именно один файл, используйте `--onefile`.

## 3. Готовый launcher

Файл: [launcher.py](launcher.py)

Он:

- находит корень проекта;
- создаёт `media` и `staticfiles`, если их нет;
- копирует `db.sqlite3` из встроенного ресурса при первом запуске, если рядом с exe базы ещё нет;
- выполняет `migrate`;
- запускает сервер без autoreload;
- ждёт, пока порт 8000 станет доступен;
- открывает браузер.
```powershell
py -3 -m PyInstaller ^
  --onefile ^
  --noconsole ^
  --name WhereIAm ^
  --add-data "static;static" ^
  --add-data "media;media" ^
  --add-data "db.sqlite3;." ^
  --add-data "mainpage\templates;templates" ^
  launcher.py
```

Для сборки в один клик используйте [build_onefile.bat](build_onefile.bat).

## 5. Команда сборки через PyInstaller

Запускать из папки проекта:

```powershell
py -3 -m PyInstaller ^
  --onedir ^
  --noconsole ^
  --name WhereIAm ^
  --add-data "static;static" ^
  --add-data "media;media" ^
  --add-data "db.sqlite3;." ^
  --add-data "mainpage\templates;templates" ^
  launcher.py
```

Если хотите ещё добавить fixtures в сборку:

```powershell
--add-data "mainpage\fixtures;fixtures"
```

## 6. Настройки static/media

В проекте уже должны быть такие настройки:

```python
STATIC_URL = '/static/'
MEDIA_URL = '/media/'
STATICFILES_DIRS = [APP_ROOT / 'static']
STATIC_ROOT = APP_ROOT / 'staticfiles'
MEDIA_ROOT = APP_ROOT / 'media'
```

Для frozen-сборки в [mysite/settings.py](mysite/settings.py) уже добавлена логика, которая ищет данные и в `APP_ROOT`, и в `sys._MEIPASS`.

## 7. Как автоматически открывать браузер

Это уже делает [launcher.py](launcher.py):

- ждёт, пока `127.0.0.1:8000` ответит;
- затем вызывает `webbrowser.open_new('http://127.0.0.1:8000')`.

## 8. Как скрыть консоль при запуске

Используйте флаг:

```powershell
--noconsole
```

Это эквивалент `--windowed`.

## 9. Как сделать portable-версию

После сборки в `dist\WhereIAm.exe` должен лежать один файл. Рядом с ним при первом запуске появятся рабочие данные:

```text
WhereIAm.exe
db.sqlite3
media\
staticfiles\
```

Эти файлы можно положить в одну папку и передавать другому человеку. Основной файл запуска только один - `WhereIAm.exe`.

Важно:

- пользователь должен запускать именно `WhereIAm.exe`;
- папка должна быть доступна на запись, чтобы SQLite и media могли обновляться;
- лучше не класть сборку в `Program Files`, если ожидаются записи в `media`.

## 10. Полная пошаговая инструкция

1. Установите Python только на компьютере сборки.
2. Перейдите в папку проекта.
3. Установите зависимости проекта:

```powershell
py -3 -m pip install -r requirements.txt
py -3 -m pip install pyinstaller Pillow
```

4. Проверьте, что `db.sqlite3`, `static` и `media` лежат в корне проекта.
5. Проверьте Django перед сборкой:

```powershell
py -3 manage.py check
```

6. Соберите проект командой PyInstaller из раздела выше.
7. Откройте `dist\WhereIAm.exe`.
8. Убедитесь, что браузер открылся на `http://127.0.0.1:8000`.
9. Проверьте загрузку фото и запись в SQLite.
10. Передайте другому человеку `WhereIAm.exe` вместе с рядом лежащими `db.sqlite3` и папкой `media`, если они уже появились после первого запуска.

## 11. Итоговая структура папки

```text
WhereIAm.exe
db.sqlite3
media\
  kras\
  sfu\
staticfiles\
```

## 12. Возможные проблемы и решения

### Сервер не запускается

- Проверьте, не занят ли порт 8000.
- Закройте другой процесс Django на этом порту.
- Посмотрите `launcher_error.log` рядом с exe.

### Браузер не открывается

- Проверьте ассоциации браузера в Windows.
- Запустите exe ещё раз.
- Если сервер стартовал, но браузер не появился, откройте `http://127.0.0.1:8000` вручную.

### Картинки не отображаются

- Убедитесь, что `static` и `templates` реально попали в папку сборки.
- Проверьте `MEDIA_ROOT` и наличие прав на запись.

### Загруженные файлы не сохраняются

- Папка сборки должна быть доступна на запись.
- Не запускайте сборку из защищённой системной папки.

### SQLite не обновляется

- Убедитесь, что `db.sqlite3` лежит рядом с exe.
- Проверьте, что файл не открыт другой программой.

### Карта или стили не грузятся

- Для Leaflet и Bootstrap проект использует внешние CDN.
- Если нужен полностью оффлайн-вариант, скачайте эти ресурсы локально и подключите из `static`.

### Слишком большой размер сборки

- Используйте `--onefile` только если вам действительно нужен один файл.
- Если размер критичен, удалите лишние файлы из `--add-data`.
- Не добавляйте лишние файлы в `--add-data`.
- Не собирайте debug-ресурсы и временные файлы.

### Нужно именно один `.exe` без папки

- Это и есть рекомендуемый режим для вашей задачи.
- Помните, что база данных и `media` должны храниться рядом с exe, чтобы изменения не терялись.
